DirectX 12 Advanced Graphics

Using DirectX 12 and HLSL to create advanced graphic techniques



Picking the framework

A key element of working on this project was selecting an API to develop the techniques in. We were given 4 choices of DirectX 11, DirectX 12, OpenGL & Vulkan. Over the years I have had experiences with all 4 APIs in some form except for DirectX 12. For Vulkan I have worked on projects in my own time, for DirectX 11 I worked on this project here and for OpenGL 1.4, this project here. So knowing that I had some experience with every other API, I choose 12 to further expand my experiences when it came to working with graphical APIs. I knew that there would be some challenges, one being that DirectX 12 gives a lot more control to the developer on how it runs, but in returns expects more from the user when it comes to managing the drawing, frames, and fences. To help myself with this project I began collecting resources such as books and websites to help develop the project.

Expandig and learning the framework

Developing using DirectX 12 was a lofty challenge, one I was happy to work towards completing. From the offset, the project materials are focused on DirectX 11 meaning that while I would have similar tasks to work on, it wouldnâ€™t be one to one and I would have to investigate how to implement each feature on its own. An example of this would be texture loading. This came as a challenge as instead of creating and then assigning a resource to a pixel shader input, it quickly became about managing heaps and how to access each texture from that heap. This added complexity was overcome by researching online and using books how DirectX 12 handles textures and how to best load them.

HLSL and Advanced Graphics

Another key element was working with HLSL to create a myriad graphics features, these include: normal mapping, parallax mapping, render to texture and shadow mapping. This was a fun and engaging challenge. This was mainly down to a slight error in one line of code in how colour was determined. This had me develop my debugging skills especially around determining what is causing issues and how to rectify them. The technique I used was splitting the sections of code into their specific parts and then iterate the design to try and locate the issue. Overall, I would say the HLSL code shows an area of development I could improve on which I am quite excited to do.
