Chat App

My experiences developing multi-threaded and networking applications



Threads, here, there, everywhere

A main element to this university project was threading. We spent our first few weeks experimenting and learning the challenges of threading, such as live lock and the philosopher’s problem. The language we used to hone our threading skills was C# due to its ability to easily manage threads and create them. Overall I really enjoyed using threads as they gave me the ability to manage multiple tasks at once in a way I hadn’t thought of before which was especially handy when it came to using WPF as it allowed me to run tasks while a thread managed the UI.

Netwroking and Graphics

Another crucial element to the project was networking. Over the period of a couple of months I learnt in detail how to set up a network, the different protocols and encryption methods and how to set up and use packets. Long with learning WPF I was able to create a functional Chat app. Another Key element was that we must add a graphical element that is synchronised across the network. I choose to make a simple breakout clone and then take that clone and implement it into the application. While it has synchronisation issues, I managed to pull this off using MonoGame
